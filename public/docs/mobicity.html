🚕 API CoopGo

Guia completo de integração para plataformas de mobilidade urbana

📖 Visão Geral

APIs públicas para integração com o sistema CoopGo

Esta documentação descreve as APIs públicas do sistema CoopGo para integração com plataformas de mobilidade urbana e aplicativos de transporte. A API permite o cadastro de empresas, autenticação, cotação de preços, solicitação e gerenciamento de corridas.

URL Base

https://api.coopgo.bygoapps.com
🔐 Segurança: Todas as requisições devem ser feitas via HTTPS. HTTP não é suportado.

🔑 Autenticação

Sistema de autenticação em duas etapas

O sistema utiliza um modelo de autenticação em duas etapas:

  1. API Key do Provider: Chave única fornecida para cada parceiro integrador
  2. Token JWT: Token temporário gerado para cada empresa cadastrada

🤝 Obtenção da API Key

Entre em contato com nossa equipe comercial para obter sua API Key de integração:

  • Email: integracao@coopgo.com.br
  • Telefone: (11) 3000-0000

1️⃣ Cadastro de Empresas

Endpoint para cadastrar empresas parceiras no sistema

POST/api/mobicity/empresas

Headers

Content-Type: application/json X-Provider-API-Key: sua_api_key_aqui

Dados Básicos da Empresa (obrigatórios)

Campo Tipo Obrigatório Validação Descrição
nome string ✅ max 100 chars Nome fantasia da empresa
cnpj string ✅ 14 dígitos CNPJ apenas números
razaoSocial string ✅ max 150 chars Razão social completa
inscricaoEstadual string ✅ max 20 chars Inscrição estadual
dataFechamento integer ✅ 1-31 Dia do fechamento mensal

2️⃣ Autenticação JWT

Geração de tokens para autenticação nas APIs

POST/api/mobicity/auth/token
⚠️ Público: Esta API não requer API Key do provider.

Campos do Request

Campo Tipo Obrigatório Descrição
empresaId UUID ✅ ID da empresa no sistema
apiKey string ✅ API Key da empresa

📤 Exemplo de Request

{ "empresaId": "123e4567-e89b-12d3-a456-426614174000", "apiKey": "empresa_1234567890abcdef" }

✅ Resposta de Sucesso (200 OK)

{ "success": true, "message": "Token gerado com sucesso", "token": "eyJhbGciOiJIUzI1NiIs...", "empresaId": "123e4567-e89b-12d3-a456-426614174000", "empresaNome": "TransPorte Rápido Ltda", "expiresIn": 36000, "tokenType": "Bearer" }

4️⃣ Solicitação de Corridas

Criar novas viagens no sistema

POST/api/v1/rides/{idCooperativa}

Headers

Content-Type: application/json Authorization: Bearer <jwt_token_da_empresa>
✅ Status Inicial: Corridas são criadas com status PENDENTE, aguardando aceite de motorista.

5️⃣ Cancelamento de Corridas

Cancelar corridas existentes

POST/api/v1/rides/cancel

Regras de Cancelamento

✅ Podem ser canceladas
PENDENTE, INICIADA, EM_ANDAMENTO
❌ Não podem ser canceladas
CANCELADA, FINALIZADA, ASSINADA

📤 Exemplo de Request

{ "rideId": "123e4567-e89b-12d3-a456-426614174000" }

✅ Resposta de Sucesso (200 OK)

{ "success": true, "message": "Viagem cancelada com sucesso", "rideId": "123e4567-e89b-12d3-a456-426614174000", "status": "CANCELADA", "canceledAt": "2024-01-15T10:30:00" }

6️⃣ Consulta de Status

Consultar status atual das corridas

GET/api/v1/rides/{rideId}/status

Status da Viagem

Status Descrição
waiting_driver Aguardando aceite de motorista
driver_assigned Motorista aceitou a corrida
driver_waiting_passenger Motorista aguardando passageiro
completed Corrida concluída
cancelled Corrida cancelada
📍 Localização em Tempo Real: O campo updatedLocation mostra a posição atual do motorista.

✅ Resposta de Sucesso (200 OK)

{ "id": "123e4567-e89b-12d3-a456-426614174000", "status": "driver_assigned", "driver": { "name": "Carlos Silva", "contact": "11999887766", "car": { "plate": "ABC-1234", "color": "Branco", "model": "Toyota Corolla" } }, "eta": 300, "updatedLocation": { "lat": -23.5495, "lng": -46.6360 } }

7️⃣ Sistema de Webhooks

Notificações automáticas de mudanças de status

O sistema envia notificações automáticas sobre mudanças de status das viagens para um endpoint webhook configurado.

⚙️ Configuração: Para receber notificações, informe a URL do seu webhook durante o cadastro da empresa ou através do suporte técnico.

Estrutura das Notificações

POST https://sua-api.com/webhooks/coopgo Content-Type: application/json Authorization: Bearer <jwt_token_empresa>

Eventos que Geram Notificações

Evento Status Descrição
Motorista aceita driver_assigned Motorista aceitou a corrida
Motorista aguarda driver_waiting_passenger Motorista chegou no local
Corrida finalizada completed Corrida foi concluída
Corrida cancelada cancelled Corrida foi cancelada
🔒 Requisitos do Webhook:
  • Responder com HTTP 200 para confirmar recebimento
  • Processar rapidamente (timeout de 5 segundos)
  • Usar HTTPS obrigatoriamente
  • Validar o token JWT no header Authorization
